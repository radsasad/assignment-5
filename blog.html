what is the document object model (dom)

The Document Object Model (DOM) is a programming interface that represents a web page's structure and allows programs to change its content, style, and structure. The DOM is a fundamental concept in web development and is used to create interactive and dynamic web pages.
Here are some key features of the DOM:
Structure
The DOM represents a web page as a logical tree, with each branch ending in a node that contains objects.
Language independence
The DOM can be implemented in various languages, but is most commonly accessed and manipulated using JavaScript.
Methods
The DOM provides methods that allow programmatic access to the tree, such as replaceChild(), appendChild(), and setAttribute()

how do you select a element from the dom?

getElementById: Selects an element by its ID. This is the simplest and fastest way to select a single element. However, since the ID must be unique to the page, this method can only be used to access one element at a time. getElementsByClassName: Selects elements by their class name. getElementsByTagName: Selects elements by their tag name. querySelector: Selects the first element that matches a specified CSS selector. querySelectorAll: Selects all elements that match a specified CSS selector.

what is event delegation in the context of the dom and why it is useful?

Event delegation is a technique in JavaScript where instead of attaching event listeners to individual child elements, you attach a single event listener to a parent element. This parent element listens for events triggered by its child elements and determines which specific child element initiated the event by inspecting the event object. This is possible because of event bubbling, where events triggered on a child element bubble up to its parent elements, allowing the parent to "catch" events that occur on its children.

It is useful because it reduces memory usage, improves performance, simplifies code, and works seamlessly with dynamically added content.
